## 1. 제품 개요

**포춘페이퍼(FortunePaper)** 는 사주(생년월일 + 성별)를 기반으로 오늘의 운세를 매일 **한 장의 리포트** 형식으로 발행하는 모바일 앱이다.

| 항목 | 내용 |
|------|------|
| 패키지명 | `com.fortune.paper` |
| 플랫폼 | Android · iOS (동시 출시) |
| 핵심 가치 | "오늘 하루를 여는 30초 리추얼" |
| 차별점 | 텍스트 나열이 아닌 **신문 1면 형식** — 날씨 등급 한 장으로 하루의 기운을 직관적으로 전달 |
| 수익 모델 | **완전 무료** (v1), 출시 후 반응 보고 유료화 검토 |
| 언어 | 한국어 (v1) |

## 2. 타겟 사용자 & 페르소나

### 페르소나 ① — 채연 (29세, 마케터)

> "출근 전에 딱 한 줄만 봐도 기분이 달라져요."

- 매일 아침 루틴이 있고, 짧고 긍정적인 콘텐츠를 선호
- 점성술·타로 앱을 써봤지만 내용이 너무 길고 광고가 많아 지침
- **원하는 것**: 알림 한 번 → 오늘의 날씨 운세 확인 → 앱 종료. 30초 이내

### 페르소나 ② — 준호 (34세, 프리랜서 디자이너)

> "하루 시작할 때 기준점 같은 게 있으면 좋겠어요."

- 규칙적인 출퇴근이 없어 하루의 리듬을 만들어줄 콘텐츠가 필요
- 앱의 시각적 완성도를 중시, UI가 촌스러우면 바로 삭제
- **원하는 것**: 매일 저녁 내일의 운세 미리 확인, 비주얼이 담긴 리포트

## 3. 핵심 사용자 경험

### 사용자 여정

```
최초 가입
 └─ 카카오 로그인
 └─ 생년월일 + 성별 입력 (온보딩)
 └─ 알림 시각 설정 (기본: 오전 8시)

매일 반복
 └─ 알림 수신 → 앱 열기
 └─ 오늘의 리포트 확인 (날씨 등급 + 한 줄 요약 + 조언)
 └─ 앱 종료

선택
 └─ 설정 → 알림 시각 변경 또는 알림 끄기
 └─ 로그아웃
```

### 골든 패스

> **알림 수신 → 앱 열기 → 오늘의 날씨 운세 한 장 확인** (목표: 30초 이내)

## 4. 기능 명세

### 4.1 카카오 로그인

| 항목 | 내용 |
|------|------|
| 방식 | 카카오 SDK → Supabase Auth 커스텀 토큰 교환 |
| 신규 유저 | 로그인 성공 → 온보딩으로 이동 |
| 기존 유저 | 로그인 성공 → 리포트 화면으로 이동 |
| 로그아웃 | 설정 화면에서 가능, 로그아웃 시 토큰 삭제 후 로그인 화면으로 이동 |

### 4.2 온보딩 (생년월일 + 성별 입력)

| 항목 | 내용 |
|------|------|
| 입력값 | 생년월일 (YYYY-MM-DD), 성별 (남/여) |
| 목적 | Claude API 사주 계산에 전달 |
| 필수 여부 | 양쪽 모두 필수, 미입력 시 다음 진행 불가 |
| 수정 | v1에서는 별도 수정 UI 없음 (탈퇴 후 재가입으로 처리) |

> **결정**: 태어난 시각(시)은 v1에서 수집하지 않는다. 미입력 시 Claude가 대표값(정오)으로 계산하도록 프롬프트 처리. 정밀 사주는 향후 유료 기능 후보.

### 4.3 오늘의 리포트 (메인 화면)

리포트는 하루 1회 생성된다. 자정(00:00 KST) 기준으로 새 날짜의 리포트가 생성되고 이전 리포트는 삭제된다.

| 구성 요소 | 내용 |
|----------|------|
| 날씨 등급 | 5단계 날씨 아이콘 + 텍스트 (대형, 한 눈에 들어와야 함) |
| 한 줄 요약 | AI가 생성하는 오늘의 종합운 헤드라인, **20자 이내** |
| 오늘의 조언 | AI의 한 마디, **50자 이내** |
| 새로고침 | 수동 새로고침 가능 (캐시 반환, 당일 재생성 없음) |
| 로딩 상태 | 첫 생성 중일 때 스켈레톤 UI 표시 |
| 오류 상태 | 생성 실패 시 안내 메시지 + 재시도 버튼 |

### 4.4 푸시 알림

| 항목 | 내용 |
|------|------|
| 기본값 | 오전 8:00 (온보딩 직후 설정 화면 안내) |
| 조정 | 분 단위 시각 지정 가능 |
| 끄기 | 알림 비활성화 가능 (`notify_time = null`) |
| 내용 | 고정 문구 + 오늘의 날씨 등급 (예: "오늘은 맑음이에요 🌤️ 리포트 확인하기") |
| 발송 방식 | Supabase pg_cron(매분) → Edge Function `notify` → FCM |

### 4.5 설정

| 항목 | 내용 |
|------|------|
| 알림 시간 변경 | 시간 피커 UI, 변경 즉시 저장 |
| 알림 끄기/켜기 | 토글 |
| 로그아웃 | 확인 다이얼로그 → 로그인 화면으로 이동 |
| 앱 버전 | 하단에 버전명 표기 |

## 5. 등급 시스템 — 날씨 5단계

운세의 강도를 날씨로 직관적으로 표현한다. **숫자·알파벳 등급 없음**.

| 코드 | 표시 | 아이콘 | 톤 가이드 |
|------|------|--------|-----------|
| `SUNNY` | 화창 | ☀️ | 적극적으로 도전하세요. 모든 게 잘 풀리는 날. |
| `CLEAR` | 맑음 | 🌤️ | 좋은 흐름이에요. 중요한 일을 오늘 처리하세요. |
| `CLOUDY` | 구름 | ☁️ | 평온한 하루. 무리하지 말고 꾸준하게. |
| `RAINY` | 비 | 🌧️ | 조심이 필요해요. 그래도 괜찮아질 거예요. |
| `STORM` | 폭풍번개 | ⛈️ | 힘든 날이지만 지나갑니다. 오늘만큼은 쉬어가도 돼요. |

**톤 원칙**

- 모든 등급에서 **위협·공포·부정적 판단 금지**. 폭풍번개도 위로와 격려로 끝낸다.
- "나쁘다"가 아닌 "조심이 필요하다"로 표현한다.
- 한국어 반말 금지 — 모든 문구는 존댓말로 작성한다.

### Claude API 응답 스펙 (업데이트)

```json
{
  "grade": "SUNNY",
  "summary": "모든 일이 술술 풀리는 날이에요",
  "advice": "망설였던 일을 오늘 시작해보세요. 타이밍이 완벽합니다."
}
```

- `grade` 허용값: `SUNNY | CLEAR | CLOUDY | RAINY | STORM`
- `summary`: 20자 이내 한국어 종합운 한 문장
- `advice`: 50자 이내 한국어 오늘의 조언

> 기존 `architecture.md`의 `grade` 허용값 (`S | A | B | C | D`)은 이 스펙으로 교체된다.

## 6. 비기능 요구사항

| 항목 | 기준 |
|------|------|
| 응답 속도 | 메인 화면 진입 후 **캐시 리포트 1초 이내** 표시 |
| 가용성 | Edge Function 실패 시 사용자 친화적 에러 메시지 + 재시도 버튼 |
| 데이터 보관 | `fortunes` 테이블 — 1일 보관. 새 날짜 리포트 생성 시 이전 레코드 삭제 |
| 개인정보 | 생년월일·성별은 운세 계산 목적으로만 사용, 제3자 제공 없음 |
| 접근성 | 날씨 등급은 아이콘만이 아닌 **텍스트(화창/맑음/...)도 병기** |
| 다국어 | 한국어 전용 (v1) |

## 7. 기술 스택 요약

| 레이어 | 기술 |
|--------|------|
| 앱 | Compose Multiplatform (KMP) |
| 인증 | 카카오 로그인 SDK + Supabase Auth |
